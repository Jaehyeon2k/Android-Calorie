# 요구사항 명세서 — 스마트 칼로리 트래커

## 1. 개요

사용자가 매일 섭취하는 음식의 칼로리를 기록하고, 목표 대사량(BMR/TDEE)과 비교하여 건강한 식습관을 유지할 수 있도록 돕는 모바일 앱이다. 서버 통신 없이 로컬 데이터베이스만으로 오프라인에서도 모든 핵심 기능이 동작하며, 식약처 공공데이터(data.go.kr) OpenAPI를 통해 음식 영양정보와 레시피를 검색하여 로컬로 복사(Import)할 수 있다.

## 2. 사용자 시나리오

### 2.1 프로필 및 목표 설정
1. 사용자가 "내 정보" 탭을 누른다.
2. 성별, 키, 몸무게, 나이, 활동 수준을 입력한다.
3. 기초대사량(BMR)과 권장 섭취 칼로리(TDEE)가 자동 계산되어 화면에 표시된다.
4. "프로필 저장" 버튼을 눌러 정보를 저장한다.

### 2.2 식단 기록 (오늘의 다이어리)
1. "다이어리" 탭에서 오늘 날짜의 섭취 현황(아침/점심/저녁/간식)을 확인한다.
2. 각 식사 시간대의 "추가" 버튼을 누르면 "음식 사전"으로 이동한다.
3. 목록에서 먹은 음식을 선택하고 섭취량(g)을 입력하여 다이어리에 등록한다.
4. 다이어리 화면 상단의 "오늘 먹은 칼로리" 프로그레스 바가 목표 대비 얼마나 채워졌는지 갱신된다.

### 2.3 음식 사전 등록 (일반/커스텀 요리)
1. "음식 사전" 탭에서 하단의 "+" 버튼을 누른다.
2. 음식명, 제공량(g), 칼로리를 직접 입력하거나, 갤러리에서 사진을 첨부한다.
3. 직접 만든 요리인 경우 "직접 만든 요리 (재료 직접 추가)" 토글을 켠다.
4. "재료 검색" 버튼을 눌러 식약처 API로 재료(예: 닭가슴살, 양파)를 검색하고 추가한다.
5. 앱이 각 재료의 무게(g)와 100g당 칼로리 데이터를 합산하여 요리의 총 칼로리를 자동 계산해 준다.
6. 저장 버튼을 누른다.

### 2.4 요리백과 API 검색 및 캐싱
1. "음식 사전" 탭에서 📚(요리백과) 버튼을 누른다.
2. 원하는 레시피명(예: "파스타")을 검색한다.
3. 식약처 API를 통해 1,000개 이상의 공식 레시피 목록이 사진과 함께 나타난다.
4. 특정 레시피의 "이 레시피 내 음식 사전에 추가" 버튼을 누른다.
5. 해당 레시피가 오프라인 DB에 영구 저장되며, 언제든 식단 기록에 사용할 수 있다.

## 3. 기능 요구사항

### 3.1 사용자 프로필 (Profile)

| ID | 요구사항 | 필수 |
|----|---------|------|
| F-PROF-01 | 사용자의 신체 정보(성별, 키, 몸무게, 나이, 활동수준)를 입력받을 수 있다 | O |
| F-PROF-02 | 신체 정보를 기반으로 BMR 및 TDEE(목표 칼로리)를 자동 계산한다 | O |
| F-PROF-03 | 앱 재실행 시 기존에 저장된 프로필 정보를 자동으로 불러온다 | O |

### 3.2 음식 사전 관리 (Food DB)

| ID | 요구사항 | 필수 |
|----|---------|------|
| F-FOOD-01 | 새로운 음식(이름, 제공량, 칼로리, 카테고리)을 수동으로 등록할 수 있다 | O |
| F-FOOD-02 | 기존에 등록된 음식의 정보를 수정 또는 삭제할 수 있다 (CRUD) | O |
| F-FOOD-03 | 음식에 갤러리 및 카메라 사진을 첨부하고 로컬에 저장하여 썸네일로 보여준다 | O |
| F-FOOD-04 | 등록된 음식들의 전체 목록을 스크롤하여 조회할 수 있다 | O |

### 3.3 레시피 및 재료 기반 요리 (Custom Recipe)

| ID | 요구사항 | 필수 |
|----|---------|------|
| F-REC-01 | 여러 재료를 조합하여 하나의 '직접 만든 요리'를 생성할 수 있다 | O |
| F-REC-02 | 재료별 중량(g)과 100g당 칼로리를 입력하면 요리의 총 칼로리를 자동 합산한다 | O |
| F-REC-03 | 0.5g 등 소수점 단위의 재료 무게를 입력해도 정밀하게 칼로리가 비례 계산된다 | O |

### 3.4 다이어리 식단 기록 (Diary)

| ID | 요구사항 | 필수 |
|----|---------|------|
| F-DIARY-01 | 특정 날짜, 식사 시간대별로 섭취한 음식을 기록할 수 있다 | O |
| F-DIARY-02 | 다이어리에 등록된 식사 기록을 삭제할 수 있다 | O |
| F-DIARY-03 | 해당 날짜에 섭취한 총 칼로리를 상단 프로그레스 바를 통해 시각적으로 보여준다 | O |
| F-DIARY-04 | 목표 칼로리(TDEE) 초과 시 프로그레스 바에 시각적 경고 색상을 표시한다 | O |

### 3.5 외부 API 연동 (Food Safety Korea)

| ID | 요구사항 | 필수 |
|----|---------|------|
| F-API-01 | 식약처 공공데이터(COOKRCP01)를 호출하여 레시피 이름으로 검색할 수 있다 | O |
| F-API-02 | 네트워크 검색 결과를 파싱하여 레시피 이름, 조리법, 썸네일 URL을 화면에 표시한다 | O |
| F-API-03 | 검색된 외부 레시피를 내 로컬 음식 사전 DB에 복사(캐싱)할 수 있다 | O |
| F-API-04 | 재료 검색 시 가공식품/튀김 등 부적합 키워드에 페널티를 주어 순수 원물을 먼저 매칭한다 | X |

## 4. 비기능 요구사항

| ID | 요구사항 |
|----|---------|
| NF-01 | 네트워크가 단절된 오프라인 상태에서도 프로필, 식단 기록, 음식 사전 조회가 정상 동작해야 한다 |
| NF-02 | 핵심 데이터는 내부 로컬 데이터베이스(Room DB)에 영구 보존된다 |
| NF-03 | Android Native 환경(Kotlin + Jetpack Compose) 기반으로 동작해야 한다 |
| NF-04 | 비동기 처리는 Kotlin Coroutines 및 Flow를 사용하여 UI 멈춤(ANR)을 방지한다 |

## 5. 화면 구성

### 5.1 오늘의 다이어리 화면 (Diary)
- 일일 총 섭취 칼로리 프로그레스 바 및 수치 표시
- 목표 칼로리 대비 남은 칼로리 표시
- 아침/점심/저녁/간식 섹션별 섭취 리스트 및 개별 삭제 버튼
- 식사 시간대별 "음식 추가" 버튼 (음식 사전으로 이동)

### 5.2 음식 사전 화면 (Food List)
- 갤러리(가로형) 형태의 저장된 음식 리스트 (이름, 칼로리, 썸네일 이미지)
- 직접 요리 추가를 위한 하단 Floating Action Button (FAB)
- 외부 요리백과 검색을 위한 별도 버튼

### 5.3 음식 추가/수정 화면 (Food Edit)
- 사진 첨부 이미지 뷰 (선택기 연동)
- 음식 이름, 카테고리, 메모 입력 텍스트 필드
- "직접 만든 요리" 스위치 및 재료 리스트 컨트롤
- 식약처 API 기반 재료 실시간 검색 다이얼로그
- 총 칼로리 표시 바 및 저장/취소 버튼

### 5.4 요리백과 API 화면 (Recipe Explorer)
- 상단 검색창 및 검색 버튼
- 로딩 인디케이터 (네트워크 지연 시)
- 레시피 썸네일, 카테고리, 요리방법이 포함된 리스트
- 개별 항목의 "내 사전에 추가(Import)" 버튼

### 5.5 내 정보 화면 (Profile)
- 성별(토글), 키, 몸무게, 나이 숫자 입력 필드
- 활동 수준 선택 드롭다운 (좌식, 가벼운 활동 등)
- 계산된 BMR 및 TDEE 결과 텍스트 영역
- 프로필 저장 버튼

## 6. 제약사항
- 공공데이터포털(식약처) API는 일일 호출량 제한이 있으므로, 무분별한 조회를 방지하기 위해 사용자가 직접 검색 버튼을 눌렀을 때만 네트워크를 호출한다.
- 사진 첨부는 Android 내부 갤러리(로컬 스토리지)에 접근 가능한 기기에서만 정상 동작한다.
