# 요구사항 명세

## 앱 개요

Calorie는 카메라를 활용해 음식 사진을 찍어 칼로리를 자동으로 인식하고, 사용자가 직접 만든 음식의 재료를 입력해 세밀하게 칼로리를 관리할 수 있는 프리미엄 Android Native 앱이다. 음식 데이터는 Room 로컬 DB에 저장하여 오프라인에서도 핵심 기능이 완벽하게 동작한다. 또한 식품의약품안전처(식품안전나라) 조리식품 레시피 공공데이터 API를 연동하여 요리 백과사전처럼 수많은 국내 레시피, 재료, 음식 이미지를 검색하고 아름다운 UI를 통해 로컬 DB에 저장할 수 있다.

## 사용자 시나리오

1. 사용자는 앱을 켜서 매우 아름답고 직관적인 대시보드에서 저장된 음식과 총 칼로리 통계를 확인한다.
2. 사용자가 카메라 버튼을 눌러 음식 사진을 찍으면, 앱이 AI Vision API를 통해 해당 음식의 종류와 예상 칼로리를 자동으로 인식하여 표시한다.
3. 만약 음식을 인식하지 못하거나 사용자가 직접 만든 새로운 음식(창작 요리)이라면, 사용자는 음식 이름과 재료(중량 포함)를 직접 입력할 수 있다.
4. 재료를 직접 입력할 경우, 앱이 각 재료의 칼로리를 합산하여 최종 칼로리를 자동으로 계산해준다.
5. 사용자는 요리 백과사전(Recipe Codex) 화면에서 식품안전나라 공공 API를 통해 국내 음식 레시피를 한글로 원활히 검색하고, 마음에 드는 레시피를 자신의 음식 사전에 즉시 저장한다.
6. 사용자는 저장된 음식 정보를 수정하거나 삭제할 수 있다. 오프라인 상태에서도 이전에 저장된 모든 기능을 제한 없이 이용 가능하다.

## 기능 요구사항

| ID | 요구사항 | 필수/구현 방향 |
| --- | --- | --- |
| F-01 | 핵심 엔티티 CRUD | **O** / `FoodItem` 생성/조회/수정/삭제 제공 |
| F-02 | 3개 이상 화면 | **O** / 목록(대시보드), 상세, 카메라 인식, 직접 등록/수정, 레시피 백과 화면 |
| F-03 | 입력 검증/예외 처리 | **O** / 잘못된 중량, 빈 이름 등에 대한 시각적 피드백(Snackbar, Error State) 제공 |
| F-04 | 데이터 영속화 | **O** / Room 로컬 DB 기반 오프라인 완벽 지원 |
| F-05 | Agent 협업 문서화 | **O** / `implementation_log.md`, `bugfix_log.md`, `subagents.md` 등 산출물 작성 |
| F-06 | AI 기반 이미지 인식 | AI Vision API 활용하여 사진에서 음식명/칼로리 도출 |
| F-07 | 국내 요리 백과사전 | 식품의약품안전처 조리식품 공공 API 연동하여 레시피 검색 및 저장 |

## 비기능 요구사항

- **프리미엄 UI/UX (Aesthetics):** 단순한 앱을 넘어 Glassmorphism, 모던 타이포그래피, 부드러운 전환 애니메이션, 다크모드 지원 등 사용자가 감탄할 수 있는 최고 수준의 유려한 디자인을 제공해야 한다.
- **오프라인 동작 보장:** 외부 AI 인식 및 레시피 검색 API를 호출할 수 없는 오프라인 상태(인터넷 단절 등)에서도 기존에 저장된 음식의 조회, 수정, 삭제, 그리고 '직접 등록' 기능은 반드시 동작해야 한다.
- **국내 공공데이터 활용:** 국내 사용자의 식습관에 완벽히 대응하기 위해 해외 API 대신 식품안전나라(식약처) 공공데이터 API를 직접 연동하여 한글 검색의 정확도와 실용성을 극대화한다.
- **구조 및 패턴:** Kotlin, Jetpack Compose(Material 3), ViewModel, Room, Navigation Compose를 활용하는 최신 Android 권장 아키텍처를 따른다.
