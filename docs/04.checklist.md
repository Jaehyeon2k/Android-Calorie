# 개발 단계 체크리스트 (Checklist) — 스마트 칼로리 트래커

전체 개발 주기를 5개의 Phase로 나누어 관리하며, 각 단계의 요구사항이 충족될 때마다 체크를 진행하였다.

## Phase 1: 기획 및 아키텍처 설계
- [x] 요구사항 명세서(`requirements.md`) 작성 및 승인
- [x] 데이터 모델 ERD(`data.md`) 설계 완료
- [x] UI/UX 와이어프레임 및 화면 구조 기획 (`index.html` 기반)
- [x] 프로젝트 초기 환경 설정 (Jetpack Compose, Room, Coil, Navigation 등)
- [x] MVVM 아키텍처 및 폴더 구조 세팅 (data, domain, ui, network)

## Phase 2: 로컬 데이터베이스 및 핵심 기능
- [x] `FoodItemEntity`, `IngredientEntity`, `MealRecordEntity` Room 테이블 생성
- [x] `CalorieDatabase` 및 `FoodDao` 구현 (오프라인 기반 CRUD)
- [x] Repository 계층 구현 및 StateFlow 연동
- [x] 기본 UI 화면 생성 (Diary, FoodList, Profile)
- [x] 네비게이션(BottomNavigationBar) 화면 이동 연동

## Phase 3: 비즈니스 로직 및 확장 기능
- [x] 식약처 (data.go.kr) OpenAPI 통신을 위한 `HttpURLConnection` 구현
- [x] API 응답 데이터 (레시피 및 영양 정보) 파싱 로직 구현
- [x] 직접 만든 요리의 재료별 칼로리 정밀 비례 계산(0.5g 등) 수식 구현
- [x] `PickVisualMediaRequest`를 이용한 갤러리 이미지 첨부 기능 연동
- [x] 로컬 캐싱: 검색한 외부 레시피를 `RecipeEntity`를 통해 내 DB로 복사하는 기능

## Phase 4: UX 개선 및 UI 폴리싱
- [x] 화면 밖 터치 시 소프트 키보드가 자연스럽게 내려가도록 글로벌 탭 제스처 적용
- [x] 목표 칼로리(TDEE) 달성률을 보여주는 상단 프로그레스 바 구현
- [x] API 검색 결과의 정확도를 높이기 위한 페널티 기반 필터링(가공/샌드위치 필터링)
- [x] 각종 에러 및 성공 피드백을 전달하는 글로벌 Snackbar 적용

## Phase 5: 안정성 확보 및 문서화 (최종)
- [x] 핵심 비즈니스 로직(단위 테스트) 및 UI 컴포넌트(Instrumented) 테스트 코드 작성
- [x] 테스트 케이스 전면 실행 및 `test_result.md`에 PASS 결과 기록 완료
- [x] 안드로이드 린트(Android Lint) 정적 검사 수행으로 코드 무결성 확인
- [x] README.md 작성 (실행, 빌드, 테스트 방법, Vibe Coding 기재)
- [x] `docs/` 내 전체 마크다운 파일 양식 및 포맷팅 통일 (프로젝트 명세 완료)
