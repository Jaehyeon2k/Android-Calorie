# 버그 수정 이력 (Bugfix Log)

## Bug 1. 칼로리 증발 현상 (0 kcal 버그)
* **현상:** 사용자가 섭취량을 '0.5g'이나 '10.5g' 같이 소수점으로 입력하면 칼로리가 0으로 저장됨.
* **원인:** 로직 내부에서 사용자 입력을 `toIntOrNull()`로 파싱하고 있었음. 이로 인해 소수점 입력 시 파싱에 실패하여 0으로 처리되거나, 정수 나눗셈에서 몫이 0이 되어 데이터가 유실됨.
* **해결:** 입력 파싱을 `toDoubleOrNull()`로 변경하고, 비율 계산식 `(kcal * currentGrams) / 100`을 모두 Double 형으로 처리한 뒤 최종 화면 표시 직전에 `.toInt()`로 변환하도록 리팩토링.

## Bug 2. 식약처 API 쓰레기 데이터 노출
* **현상:** '닭가슴살' 검색 시 100g당 240kcal로 터무니없는 수치가 자동 입력됨.
* **원인:** 식약처 API에서 `numOfRows=1`로 요청하여 무조건 최상단 결과 1개만 받았는데, 첫 결과가 '샌드위치_닭가슴살'이었음.
* **해결:** `numOfRows=20`으로 늘린 후, `name.length`가 가장 짧으면서 "튀김/샌드위치" 등 가공 키워드가 없는 가장 순수한 식재료를 찾는 스코어링 필터 알고리즘을 적용.

## Bug 3. API 검색 결과가 없는 경우 처리 미흡
* **현상:** '아무음식' 등 사전에 없는 단어 입력 시, 이전 칼로리 값이 그대로 남아있음.
* **원인:** API 반환값이 `null`일 때 UI State(manualCalories)를 갱신하지 않고 무시함.
* **해결:** `info == null` 분기문을 추가하여, 일치하는 데이터가 없을 시 수동 입력 칼로리 칸을 "0"으로 리셋하도록 강제 초기화 로직 구현.
