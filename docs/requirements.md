# 요구사항 명세 (Requirements)

## 1. 개요 (Overview)
* **앱 이름:** 스마트 칼로리 트래커 (Smart Calorie Tracker)
* **목표:** 사용자가 매일 섭취하는 음식의 칼로리를 기록하고, 목표 대사량(BMR/TDEE)과 비교하여 건강한 식습관을 유지할 수 있도록 돕는 오프라인 기반 Android Native 애플리케이션.
* **주요 타겟:** 다이어터, 식단 관리자, 바디프로필 준비자.

## 2. 사용자 시나리오 (User Scenario)
1. **사용자 프로필 설정:** 사용자는 앱에 최초 접속하거나 프로필 탭에서 성별, 키, 몸무게, 나이, 활동 수준을 입력하여 권장 일일 섭취 칼로리를 계산받는다.
2. **음식 검색 및 등록:** 사용자는 오늘 먹은 '닭가슴살'을 검색한다. 식약처 공공데이터(data.go.kr) API를 통해 정확한 칼로리가 자동 입력된다.
3. **직접 만든 음식 (레시피) 기록:** 샐러드 파스타를 만들어 먹은 사용자는 '직접 만든 음식' 토글을 켜고, 샐러드, 파스타면, 올리브오일의 각 그램수를 입력한다. 앱이 총 칼로리를 자동 계산해 준다. 카메라/갤러리에서 완성된 요리 사진을 첨부한다.
4. **다이어리 확인:** 다이어리 화면에서 오늘 아침, 점심, 저녁, 간식으로 먹은 칼로리의 총합을 보고, 권장 칼로리 대비 남은 칼로리를 파악한다.

## 3. 기능 요구사항 (Functional Requirements)
* **F-01 (프로필 관리):** 사용자의 신체 정보를 기반으로 BMR 및 TDEE 자동 계산.
* **F-02 (음식 데이터 CRUD):** 일반 음식 및 직접 만든 음식(재료 포함)의 생성/조회/수정/삭제. (사진 첨부 기능 포함)
* **F-03 (다이어리 기록 CRUD):** 날짜별, 식사 시간대별(아침/점심/저녁/간식) 섭취 기록 추가/수정/삭제.
* **F-04 (API 연동):** 식약처 공공데이터(data.go.kr) 및 요리백과 API를 연동하여 음식 영양정보 및 레시피 자동 검색 및 정확도 기반 필터링 (튀김, 가공식품 배제 로직).
* **F-05 (소수점 정밀 계산):** 0.5g 등 소수점 단위의 재료 섭취량에도 칼로리가 유실되지 않도록 Double 타입 기반 정밀 계산 제공.

## 4. 비기능 요구사항 (Non-Functional Requirements)
* **오프라인 퍼스트:** Room DB를 사용하여 인터넷이 끊긴 상태에서도 기존에 등록된 음식 및 식단 기록을 완벽하게 조회/추가 가능.
* **UX/UI:** Jetpack Compose 기반의 선언형 UI, Material 3 가이드라인 준수. 화면 밖 터치 시 키보드 숨김 등 사용성 극대화.
* **아키텍처:** MVVM 패턴 및 Kotlin Coroutines/Flow 기반 비동기 처리.
