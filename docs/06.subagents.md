# Agent 협업 방식

## 역할 분담

| 역할 | 담당 |
| --- | --- |
| 기획 Agent | 과제 요구사항 분석, 프리미엄 앱 타겟 및 완벽한 기능 명세 구체화 |
| 설계 Agent | Room 데이터 모델, AI 인식-수동 입력 간의 Fallback 흐름, 아름다운 UI 화면 설계 |
| 구현 Agent | Kotlin/Jetpack Compose 코딩, 아름다운 UI 컴포넌트(Glassmorphism 등) 구현 |
| API Agent | TheMealDB API(레시피) 및 외부 AI Vision API(사진 칼로리 인식) 연동, 오류 처리 |
| 테스트 Agent | 단위 테스트, Room 테스트, UI 테스트 계획 수립 |
| 문서화 Agent | docs 산출물, 진행 과정 및 트레이드오프, 완벽한 문서 정합성 유지 |

## 협업 기록

- 사용자와 Agent가 초기 논의를 거쳐, 단순한 CRUD 앱을 넘어서는 완벽하고 아름다운 '프리미엄 칼로리 사전 앱'을 만들기로 합의했다.
- 사용자의 강력한 요청에 따라, 카메라로 사진을 찍었을 때 외부 AI를 사용해 즉각 칼로리를 도출하는 기능을 앱의 핵심 흐름으로 편입시켰다.
- AI가 인식을 못하거나 직접 창작한 요리일 경우, '재료별 칼로리 합산'이라는 정교한 수동 입력 기능을 만들어 오프라인에서도 작동하도록 설계했다.
- 요리 백과사전 기능을 위해 TheMealDB 외부 API를 도입하고, 온라인에서 검색한 멋진 레시피와 이미지를 오프라인 Room DB에 저장하여 시각적인 완성도를 높였다.
- AI 사용이 허락된 과제인 만큼 앱의 UI를 단순한 형태에서 벗어나 고해상도 이미지, 모던 타이포그래피, 애니메이션이 들어간 완벽한 수준으로 끌어올리기로 최종 결정하고 문서를 전면 수정하였다.
