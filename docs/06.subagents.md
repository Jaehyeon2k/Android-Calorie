# Agent 역할 분담 및 협업 (Subagents & AI Collaboration) — 스마트 칼로리 트래커

본 과제는 개발자인 인간(USER)과 AI 코딩 어시스턴트(Antigravity Agent)가 긴밀하게 협업하는 **바이브 코딩(Vibe Coding)** 방식을 적극 채택하여, 기획부터 설계, 구현, 디버깅, 테스트, 산출물 작성까지 전 과정을 공동으로 진행하였습니다.

## 1. 역할 분담 (Role Distribution)

| 주체 | 역할 및 수행 내용 |
|------|-------------------|
| **USER** (인간 개발자) | - **도메인 요구사항 정의:** 앱의 목표(다이어트 트래킹) 및 사용자 니즈 구체화<br>- **아키텍처 결정 승인:** Agent가 제안한 DB 구조 및 프레임워크 스택 최종 승인<br>- **API Key 획득:** 식약처 및 공공데이터포털 등 외부 서비스 접근 권한 확보<br>- **QA 및 사용성 평가:** 에뮬레이터에서 실제 빌드 후 UI/UX 피드백 제공 |
| **AI Agent** (Antigravity) | - **코드 구현 (Jetpack Compose & Room):** UI 컴포넌트, DAO, 비동기 로직 등 전체 보일러플레이트 및 코어 로직 작성<br>- **네트워크 통신망 구축:** `HttpURLConnection` 기반의 경량 API 통신 및 JSON 파싱 로직 설계<br>- **실시간 디버깅:** 컴파일 에러 및 로직 오류 발생 시 즉각적인 원인 분석 및 패치 적용<br>- **자동화 문서 작성:** 요구사항 명세서, ERD, 체크리스트 등 마크다운 프로젝트 산출물 정형화 |

## 2. 주요 협업 성과 및 시너지

### 2.1 API 한계 극복 및 피봇(Pivot) 주도
- **이슈:** 초기에는 글로벌 식품 DB인 'FatSecret API' 도입을 고려했으나, 한국어 미지원 및 무료 버전에 제약이 있음을 파악함.
- **AI 협업:** Agent의 제안에 따라 즉각 **식약처(Food Safety Korea) 공공데이터 OpenAPI로 피봇(Pivot)** 하였음. 또한 공공데이터 특성상 가공식품이 상단에 노출되는 문제를 해결하기 위해, Agent가 즉석에서 "페널티 스코어링(Penalty Scoring)" 알고리즘을 고안하고 구현하여 검색 품질을 획기적으로 개선함.

### 2.2 선제적 데이터 타입 마이그레이션 (Refactoring)
- **이슈:** 사용자 피드백(USER QA) 중 "10.5g처럼 소수점 단위로 무게를 입력하면 칼로리가 증발(0)한다"는 치명적 버그가 리포트됨.
- **AI 협업:** Agent가 즉시 원인(정수형 캐스팅에 의한 데이터 유실)을 파악하고, DB Entity부터 UI 계층(ViewModel)까지 전체 도메인을 관통하는 `Int -> Double` 마이그레이션을 단 한 번의 커밋으로 무결점 처리함.

### 2.3 오프라인 퍼스트(Offline-First) 설계
- **이슈:** 인터넷이 안 되는 환경에서도 앱이 동작해야 한다는 요구사항 충족 필요.
- **AI 협업:** 단순 API 조회가 아닌, 검색된 레시피를 로컬 `RecipeEntity`와 `FoodItemEntity`에 영구 보존(Import)하는 더블캐싱 전략을 Agent가 제안하고 성공적으로 구현함.

## 3. 결론
인간 개발자의 도메인 이해/의사결정 능력과 AI Agent의 빠르고 정확한 코드 작성 능력이 시너지를 발휘하여, 짧은 개발 기간 내에 높은 완성도와 무결점을 자랑하는 프로덕션 레벨의 Android Native 앱을 완성할 수 있었습니다.
