# 테스트 계획 (Test Plan) — 스마트 칼로리 트래커

## 1. 테스트 전략

본 프로젝트는 비즈니스 로직의 정확성, UI 컴포넌트의 안정성, 기기 권한 처리를 모두 포함하는 다각적 테스트 전략을 채택한다.

1. **단위 테스트 (Unit Test, JUnit4)**
   - **대상:** 칼로리 비례 계산 로직, 재료 합산 로직, 기초대사량(BMR) 산출 공식.
   - **목적:** 수식의 수학적 무결성, 특히 0.5g 등 소수점 데이터 처리 시 오류 여부 검증.
   - **실행 방법:** `./gradlew test`

2. **UI 테스트 (Instrumented Test, Compose Test Rule)**
   - **대상:** Navigation 라우팅, 스낵바 메시지, 터치 이벤트.
   - **목적:** 빈 폼 제출 시 경고, 키보드 자동 숨김 등 UI/UX 명세 준수 확인.
   - **실행 방법:** `./gradlew connectedAndroidTest`

3. **정적 분석 (Static Analysis)**
   - **대상:** 안드로이드 전체 코드.
   - **목적:** 잠재적 메모리 릭, 사용하지 않는 리소스, 구조적 오류 사전 탐지.
   - **실행 방법:** `./gradlew lint`

## 2. 주요 테스트 케이스 (TC)

### 2.1 핵심 비즈니스 로직 단위 검증
| TC-ID | 기능 분류 | 테스트 시나리오 | 예상 결과 |
|-------|---------|----------------|-----------|
| TC-UNIT-01 | 칼로리 비례 계산 | 100g당 120kcal인 음식을 50.5g 섭취했다고 입력 | 비례 계산되어 60kcal(반올림/내림)로 저장됨 |
